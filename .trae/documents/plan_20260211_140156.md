## 替换马头像为自定义图片的实现计划

### 任务分析

用户希望将所有马的头像替换为自己定义的图片，需要：

1. 创建图片目录，存放用户自定义的图片
2. 修改代码，支持从图片文件加载头像，而不是使用SVG
3. 确保不破坏现有样式和排版
4. 提供清晰的替换规则

### 实现方案

#### 步骤1：创建图片目录结构

* 在项目根目录创建`public/horse-avatars/`目录

* 目录结构：

  ```
  public/
  └── horse-avatars/
      ├── 1.png   # 纯血牛马
      ├── 2.png   # 极品脆皮马
      ├── 3.png   # 假装在跑马
      ├── 4.png   # 被牵着走马
      ├── 5.png   # 想逃没草马
      ├── 6.png   # 躺平心虚马
      ├── 7.png   # 半退休马
      ├── 8.png   # 已读不回马
      ├── 9.png   # AI 边缘马
      ├── 10.png  # 情绪外包马
      └── 11.png  # 天生拽马
  ```

#### 步骤2：修改HorseAvatars组件

* 修改`components/HorseAvatars.tsx`文件

* 将SVG组件替换为图片加载组件

* 保持相同的接口和样式

#### 步骤3：更新相关代码

* 修改`App.tsx`中使用马头像的地方

* 确保所有使用马头像的地方都能正确加载新的图片

#### 步骤4：样式和排版调整

* 确保图片大小与原SVG一致

* 保持原有的布局和间距

* 确保响应式设计正常工作

### 技术实现细节

#### 图片加载组件

```typescript
// 新的HorseAvatar组件
const HorseAvatar = ({ type, className }: { type: string, className?: string }) => {
  const imagePath = `/horse-avatars/${type}.png`;
  
  return (
    <div className={`relative overflow-hidden ${className}`}>
      <img 
        src={imagePath} 
        alt={`Horse ${type}`} 
        className="w-full h-full object-contain"
        onError={(e) => {
          // 图片加载失败时的 fallback
          console.error(`Failed to load horse avatar: ${imagePath}`);
        }}
      />
    </div>
  );
};
```
